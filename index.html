Prework Study Guide

HTML

  • HTML provides structure/frame of webpage
  • The head element contains information about the webpage
  • The body element represents the visible content shown to he user
  • Header order matters for screen readers and devices. H1 always first and once per page
  • When using image alt= is descriptive text used by screen readers/devices to describe the image
  • declarations are not case sensitive but elements ARE case sensitive

CSS

  • CSS provides the style of a webpage
  • a period indicates a class selector .classname
  • 3 ways to style:
    1. inline - style dirctly in HTML, quick changes but less flexible, must edit individually
    2. internal style sheet - rules for webpage in head section, apply only to that page can style multiple elements
    3. external style sheet - most preferred, all css rules in seperate file, easier design changes, link element placed in head section
  • Seperation of concerns - seperate code into diff files based on use. Each section should have own responsibility
  • CSS rule structure: Declaration = (propertty: value;)
  • CSS BOX MODEL: content, padding,border,margin
    1. margin: indicates how much space we want around the outside of an element
    2. padding: indicates how much space we want around the content inside an element
    3. increasing the padding makes entire element larger
  • *xxxxx= wild card symbol - applies rule to all elements visible on page
  • can group selectors together to use same declaration and styling
  • CSS file in linked to HTML file
  • a period .classname indicates a class selector, allows rule to be shared to any element and can be assigned in HTML using class="classname"
  • assign colors with words or HEX# color codes

Git

  • git is used to track changes
  • git status: checks what branch you are currently in ALWAYS check this before working
  • git checkout -b branchname: created a new branch and switches to it
  • git Add -A: adds mods in current working branch to staging area, -A all files on working branch
  • git commit -m "comment": adds to repository
  • git pull branchname (ex. git pull main): syncs any modifications to your version
  • it is imporant frequently sync the local version of code with the remote version on GitHub
  • touch filename to create a file
